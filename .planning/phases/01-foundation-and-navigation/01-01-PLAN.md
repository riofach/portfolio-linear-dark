# Phase 1: Foundation Setup

## Goal
Initialize the Next.js 16 architecture with Tailwind CSS v4, Motion v12, and the project data structure.

## Context
This is the "Bleeding Edge" setup. We are using Tailwind v4 (CSS-first config) and Motion v12 (formerly framer-motion). We also need to set up the data layer early so UI components have content to consume.

## Tasks

<task>
<instruction>
Initialize a new Next.js 16 project in the current directory.
1. Run `npx create-next-app@latest . --typescript --eslint --tailwind --app --src-dir --import-alias "@/*" --use-npm`
   - If the directory is not empty, you might need to use a temp dir and move files, or just ensure we don't overwrite critical config if it exists. (Since we are in the root, check if package.json exists first).
   - Actually, looking at the tools, we should just run the install commands if the app is already scaffolded, or scaffold if not.
   - **Crucial:** Ensure we are using Tailwind v4. Standard create-next-app might default to v3.
   - If `create-next-app` installs Tailwind v3, we must upgrade it immediately.
</instruction>
<verification>
- `package.json` exists.
- `dependencies` includes `next`, `react`, `react-dom`.
</verification>
</task>

<task>
<instruction>
Upgrade/Install dependencies for the "Linear Dark" stack.
1. Uninstall `tailwind` v3 packages if present.
2. Install Tailwind v4: `npm install tailwindcss @tailwindcss/postcss postcss`
3. Install Animation & Scroll: `npm install motion lenis`
4. Install Utilities: `npm install clsx tailwind-merge lucide-react`
</instruction>
<verification>
- `package.json` shows `tailwindcss` version 4.x (or equivalent beta/canary if standard is not out, but research says v4 is target).
- `motion` is installed (not `framer-motion`).
</verification>
</task>

<task>
<instruction>
Configure Tailwind CSS v4.
1. Update `src/app/globals.css`:
   - Use `@import "tailwindcss";`
   - Define the theme variables under `@theme`.
   - Add the "Linear Dark" specific colors:
     - `--color-background`: `var(--color-zinc-950)`
     - `--color-foreground`: `#ffffff`
     - Custom fonts if needed (Geist is default in Next.js now, ensure it maps to `--font-sans`).
</instruction>
<verification>
- `src/app/globals.css` contains `@import "tailwindcss";`
- No `tailwind.config.ts` file should be needed for basic v4 usage (or it should be minimal).
</verification>
</task>

<task>
<instruction>
Configure the Root Layout.
1. In `src/app/layout.tsx`:
   - Import `Geist` from `next/font/google`.
   - Configure the font variable.
   - Import `ReactLenis` from `lenis/react` (or `@studio-freight/react-lenis` if the new package isn't resolving, but research says `lenis`).
   - Wrap the children in `<ReactLenis root>`.
   - Add the `antialiased` utility to the body.
</instruction>
<verification>
- `src/app/layout.tsx` imports `lenis`.
- Layout wraps content in smooth scroll provider.
</verification>
</task>

<task>
<instruction>
Create the Utility library.
1. Create `src/lib/utils.ts`.
2. Export the standard `cn` function using `clsx` and `tailwind-merge`.
</instruction>
<verification>
- `src/lib/utils.ts` exists and exports `cn`.
</verification>
</task>

<task>
<instruction>
Create the Data Layer with specific schema requirements.
1. Create `src/data`.
2. Create `src/data/config.ts`: Export `siteConfig` object containing:
   - `name`: string
   - `description`: string
   - `socials`: Object containing keys for `github`, `linkedin`, `instagram`, `threads`, `dribbble`.
3. Create `src/data/skills.ts`: Export `skills` array (strings).
4. Create `src/data/projects.ts`: Export `projects` array.
   - **Crucial:** The `Project` interface must include:
     - `title`: string
     - `description`: string
     - `techStack`: string[]
     - `demoUrl`: string (optional/nullable)
     - `repoUrl`: string (optional/nullable)
     - `image`: string (path to asset)
5. Create `src/data/experience.ts`: Export `experience` array (Role, Company, Date, Description).
</instruction>
<verification>
- `src/data/config.ts` includes the required 5 social platforms.
- `src/data/projects.ts` specifically includes `demoUrl` and `repoUrl` fields in the type definition.
</verification>
</task>

## Must Haves
- Next.js 16 + Tailwind v4 working (CSS compiles).
- `motion` library installed.
- Global smooth scroll (Lenis) configured in Layout.
- Data files exist and match the specific schema (including `demoUrl`, `repoUrl` and specific socials).
