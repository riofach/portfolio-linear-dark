# Core Schemas

Define the primary content types for the portfolio.

## Tasks

### 1. Create Project Schema
Define the schema for portfolio projects.
-   Create `sanity/schemas/documents/project.ts`.
-   Fields: title, slug, thumbnail, description (localized), tags, demoUrl, repoUrl.

### 2. Create Experience Schema
Define the schema for work experience.
-   Create `sanity/schemas/documents/experience.ts`.
-   Fields: company, role, startDate, endDate, current, description (localized), logo.

### 3. Create Skill Schema
Define the schema for technical skills.
-   Create `sanity/schemas/documents/skill.ts`.
-   Fields: name, icon, category (frontend, backend, tools), proficiency.
